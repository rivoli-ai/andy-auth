Installation Guide

This guide walks you through installing Andy Auth from source and getting it running on your local development environment.

Quick Start Available

If you want to get started quickly, check out the Quick Start guide which provides the fastest path to a running instance.

Prerequisites

Before installing Andy Auth, ensure you have the following software installed:

.NET 8 SDK

Andy Auth is built on .NET 8. Download and install the SDK from the official Microsoft website:

bash 
# Verify .NET installation
dotnet --version
# Should output: 8.0.x or higher

# If not installed, download from:
# https://dotnet.microsoft.com/download/dotnet/8.0

PostgreSQL 14+

Andy Auth uses PostgreSQL for data storage. Install PostgreSQL 14 or later:

bash 
# macOS (using Homebrew)
brew install postgresql@14
brew services start postgresql@14

# Ubuntu/Debian
sudo apt update
sudo apt install postgresql postgresql-contrib

# Windows
# Download installer from https://www.postgresql.org/download/windows/

# Verify installation
psql --version
# Should output: psql (PostgreSQL) 14.x or higher

Git

Git is required to clone the repository:

bash 
# Verify Git installation
git --version
# Should output: git version 2.x.x
Requirement Minimum Version Recommended
.NET SDK 8.0 8.0 (latest)
PostgreSQL 14 16
Git 2.0 2.40+
RAM 2 GB 4 GB
Disk Space 500 MB 1 GB

Cloning the Repository

Clone the Andy Auth repository from GitHub:

bash 
# Clone the repository
git clone https://github.com/rivoli-ai/andy-auth.git

# Navigate to the project directory
cd andy-auth

# View the project structure
ls -la
# You should see:
# - src/           (source code)
# - tests/         (test projects)
# - docs/          (documentation)
# - Andy.Auth.sln  (solution file)

Database Setup

Create a PostgreSQL database for Andy Auth:

bash 
# Connect to PostgreSQL
psql -U postgres

# Create a new database
CREATE DATABASE andyauth;

# Create a user for the application
CREATE USER andyauth_user WITH ENCRYPTED PASSWORD 'your_secure_password';

# Grant privileges
GRANT ALL PRIVILEGES ON DATABASE andyauth TO andyauth_user;

# Exit psql
\q
Use Strong Passwords

Replace your_secure_password with a strong, unique password. Never use the example password in production environments.

Apply Database Migrations

Andy Auth uses Entity Framework Core for database management. Apply migrations to create the schema:

bash 
# Navigate to the server project
cd src/Andy.Auth.Server

# Update the connection string in appsettings.Development.json first
# Then apply migrations
dotnet ef database update

# Or run all migrations explicitly
dotnet ef migrations list
dotnet ef database update

Environment Configuration

Configure the application for your environment:

Development Configuration

Create or update appsettings.Development.json:

json 
{
    "ConnectionStrings": {
        "DefaultConnection": "Host=localhost;Database=andyauth;Username=andyauth_user;Password=your_secure_password"
    },
    "Logging": {
        "LogLevel": {
            "Default": "Information",
            "Microsoft.AspNetCore": "Warning",
            "OpenIddict": "Debug"
        }
    },
    "OpenIddict": {
        "Issuer": "https://localhost:7088"
    }
}

User Secrets (Recommended for Development)

Use .NET User Secrets to store sensitive configuration:

bash 
# Initialize user secrets
cd src/Andy.Auth.Server
dotnet user-secrets init

# Set the connection string
dotnet user-secrets set "ConnectionStrings:DefaultConnection" \
    "Host=localhost;Database=andyauth;Username=andyauth_user;Password=your_secure_password"

# View all secrets
dotnet user-secrets list

Running Locally

Start the application in development mode:

bash 
# From the src/Andy.Auth.Server directory
dotnet run

# Or with hot reload enabled
dotnet watch run

# The application will start on:
# - HTTPS: https://localhost:7088
# - HTTP:  http://localhost:5271

Verify Installation

Check that the application is running correctly:

bash 
# Check the OpenID discovery endpoint
curl https://localhost:7088/.well-known/openid-configuration

# You should see a JSON response with endpoints like:
# - authorization_endpoint
# - token_endpoint
# - userinfo_endpoint
# - registration_endpoint

You can also visit the following URLs in your browser:

URL Description
https://localhost:7088 Application home page
https://localhost:7088/docs Documentation
https://localhost:7088/.well-known/openid-configuration OpenID Discovery
https://localhost:7088/admin Admin dashboard

Creating the First Admin User

Create an administrator account to access the admin dashboard:

bash 
# The first user registered automatically becomes an admin
# Navigate to https://localhost:7088/Account/Register and create an account

# Alternatively, use the CLI tool:
cd src/Andy.Auth.Server
dotnet run -- create-admin --email admin@example.com --password "YourSecurePassword123!"

Docker Installation (Alternative)

You can also run Andy Auth using Docker:

bash 
# Build the Docker image
docker build -t andy-auth .

# Run with Docker Compose
docker-compose up -d

# Or run standalone with environment variables
docker run -d \
    -p 5001:443 \
    -e ConnectionStrings__DefaultConnection="Host=host.docker.internal;Database=andyauth;Username=andyauth_user;Password=your_secure_password" \
    -e ASPNETCORE_URLS="https://+:443" \
    -e ASPNETCORE_Kestrel__Certificates__Default__Path="/https/cert.pfx" \
    -e ASPNETCORE_Kestrel__Certificates__Default__Password="cert_password" \
    andy-auth

Troubleshooting

Common Issues

Issue Solution
Database connection failed Verify PostgreSQL is running and connection string is correct
SSL certificate errors Trust the development certificate: dotnet dev-certs https --trust
Port already in use Change the port in launchSettings.json or stop the conflicting process
Migration errors Ensure EF tools are installed: dotnet tool install --global dotnet-ef

Getting Help

If you encounter issues not covered here:

  • Check the GitHub Issues for similar problems
  • Review the application logs in the console output
  • Enable debug logging by setting Logging:LogLevel:Default to Debug
