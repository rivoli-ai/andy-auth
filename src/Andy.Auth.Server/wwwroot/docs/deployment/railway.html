Deploying to Railway

Railway is the primary deployment target for Andy Auth. It provides automatic HTTPS, PostgreSQL databases, and zero-config deployments from your Git repository.

Why Railway?

Railway offers a seamless deployment experience with automatic SSL certificates, built-in PostgreSQL support, and environment variable management. Andy Auth is pre-configured with railway.json and nixpacks.toml for optimal deployment.

Prerequisites

  • A Railway account (free tier available)
  • Andy Auth source code in a Git repository (GitHub, GitLab, or Bitbucket)
  • Basic understanding of environment variables

Quick Deploy

Create a New Project

Log into Railway and click "New Project". Select "Deploy from GitHub repo" and connect your repository.

Add PostgreSQL Database

In your project, click "New" and select "Database" then "PostgreSQL". Railway will provision a database and set the DATABASE_URL automatically.

Configure Environment Variables

Add the required environment variables in the "Variables" tab. See the configuration section below.

Deploy

Railway will automatically build and deploy your application using the Dockerfile.

Configuration Files

railway.json

Andy Auth includes a pre-configured railway.json that specifies the build and deploy settings:

json 
{
  "$schema": "https://railway.app/railway.schema.json",
  "build": {
    "builder": "DOCKERFILE",
    "dockerfilePath": "Dockerfile"
  },
  "deploy": {
    "restartPolicyType": "ON_FAILURE",
    "restartPolicyMaxRetries": 10
  }
}

nixpacks.toml (Alternative)

For builds without Docker, Railway can use Nixpacks. The included nixpacks.toml configures a self-contained .NET 8 build:

toml 
[phases.setup]
nixPkgs = ["dotnet-sdk_8"]

[phases.build]
cmds = ["dotnet publish src/Andy.Auth.Server/Andy.Auth.Server.csproj -c Release -o out --self-contained true -r linux-x64"]

[start]
cmd = "./out/Andy.Auth.Server"

Environment Variables

Configure these environment variables in Railway's "Variables" tab:

Variable Required Description
ConnectionStrings__DefaultConnection Required PostgreSQL connection string. Use $DATABASE_URL reference or copy from Railway PostgreSQL service.
ASPNETCORE_ENVIRONMENT Required Set to Production for production deployments.
OpenIddict__UseEphemeralKeys Required Set to true for Railway deployments (tokens invalidate on restart).
CorsOrigins__AllowedOrigins__0 Optional First allowed CORS origin (e.g., https://myapp.com).
CorsOrigins__AllowedOrigins__1 Optional Second allowed CORS origin. Add more with incrementing indices.
AzureAd__ClientId Optional Azure AD/Microsoft Entra ID client ID for external authentication.
AzureAd__ClientSecret Optional Azure AD client secret.
AzureAd__TenantId Optional Azure AD tenant ID. Use common for multi-tenant.
DynamicClientRegistration__RequireInitialAccessToken Optional Set to true to require tokens for DCR (recommended for production).
Ephemeral Keys Warning

When using OpenIddict__UseEphemeralKeys=true, all tokens are invalidated when the application restarts. This is acceptable for UAT/staging but consider using persistent keys for production (see Production Checklist).

Database Setup

Using Railway PostgreSQL

Railway provides managed PostgreSQL databases. The connection string is automatically available as DATABASE_URL:

text 
# Railway provides this automatically:
DATABASE_URL=postgresql://user:password@host:port/database

# Andy Auth converts this URI format automatically to:
Host=host;Port=port;Database=database;Username=user;Password=password

Set the connection string variable using Railway's reference syntax:

text 
ConnectionStrings__DefaultConnection=${{Postgres.DATABASE_URL}}

Database Migrations

Andy Auth automatically runs database migrations on startup. No manual migration steps are required.

SSL/HTTPS Configuration

Railway automatically provisions SSL certificates for your domain. Andy Auth is configured to work behind Railway's HTTPS proxy:

  • Forwarded Headers - Andy Auth reads X-Forwarded-For and X-Forwarded-Proto headers
  • HTTP Internally - The app runs on HTTP internally; Railway handles SSL termination
  • PORT Variable - Railway sets the PORT environment variable automatically
text 
Internet (HTTPS) -> Railway Proxy -> Andy Auth (HTTP on $PORT)

Custom Domain

To use a custom domain with Railway:

  1. Go to your service's "Settings" tab
  2. Under "Domains", click "Add Custom Domain"
  3. Enter your domain (e.g., auth.mycompany.com)
  4. Add the provided CNAME record to your DNS
  5. Railway will automatically provision an SSL certificate

Health Checks

Andy Auth exposes endpoints that can be used for health monitoring:

Endpoint Method Description
/.well-known/openid-configuration GET OpenID Connect discovery document - confirms server is operational
/.well-known/jwks GET JSON Web Key Set - confirms signing keys are available
/ GET Home page - basic availability check

Monitoring and Logs

Railway provides built-in logging and monitoring:

  • Logs - View real-time logs in the "Deployments" tab
  • Metrics - CPU, memory, and network metrics available in the dashboard
  • Alerts - Configure deployment notifications in project settings

Troubleshooting

Build Fails

If the Docker build fails, check:

  • The Dockerfile is in the repository root
  • All project files are committed to Git
  • The .NET SDK version matches (8.0)

Database Connection Errors

If you see connection errors:

  • Verify ConnectionStrings__DefaultConnection is set correctly
  • Check that the PostgreSQL service is running in your Railway project
  • Ensure the database URL format is correct (Railway provides postgresql:// format)

HTTPS/Redirect Issues

If OAuth redirects fail:

  • Ensure ASPNETCORE_ENVIRONMENT is set to Production
  • Verify your redirect URIs use HTTPS
  • Check CORS origins are configured correctly
