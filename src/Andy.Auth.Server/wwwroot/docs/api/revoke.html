Revocation Endpoint

The revocation endpoint allows clients to invalidate access tokens and refresh tokens. This implements RFC 7009 (OAuth 2.0 Token Revocation).

POST /connect/revoke
When to Revoke Tokens

Revoke tokens when a user logs out, when a session needs to be terminated, when a device is lost or compromised, or when a user disconnects an MCP integration.

Request

Request Parameters

Parameter Required Description
token Required The token to revoke (access token or refresh token)
token_type_hint Optional Hint about the token type: access_token or refresh_token
client_id Required The client identifier
client_secret Conditional Required for confidential clients

Request Example

http 
POST /connect/revoke HTTP/1.1
Host: auth.example.com
Content-Type: application/x-www-form-urlencoded

token=8xLOxBtZp8&
token_type_hint=refresh_token&
client_id=my-client

For Confidential Clients

http 
POST /connect/revoke HTTP/1.1
Host: auth.example.com
Content-Type: application/x-www-form-urlencoded
Authorization: Basic bXktY2xpZW50Om15LXNlY3JldA==

token=eyJhbGciOiJSUzI1NiIsInR5cCI6ImF0K2p3dCJ9...&
token_type_hint=access_token

Response

The revocation endpoint returns 200 OK whether the token was successfully revoked, was already invalid, or was unknown. This prevents information leakage about token validity.

http 
HTTP/1.1 200 OK
Cache-Control: no-store
Pragma: no-cache
Always Returns 200

Per RFC 7009, the server returns 200 OK even if the token was already invalid or didn't exist. This prevents attackers from determining if a token was valid by observing the response.

Revocation Cascade

When you revoke a token, the following cascade behavior applies:

Token Revoked Effect
Refresh Token The refresh token and all access tokens issued from it are revoked
Access Token Only the specific access token is revoked; refresh token remains valid
Best Practice

When logging a user out, revoke the refresh token to ensure all associated access tokens are also invalidated. This provides a complete logout experience.

Code Examples

JavaScript - Logout Implementation

javascript 
async function logout() {
  const refreshToken = localStorage.getItem('refresh_token');

  if (refreshToken) {
    // Revoke the refresh token (which also revokes all associated access tokens)
    await fetch('https://auth.example.com/connect/revoke', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/x-www-form-urlencoded'
      },
      body: new URLSearchParams({
        token: refreshToken,
        token_type_hint: 'refresh_token',
        client_id: 'my-client'
      })
    });
  }

  // Clear local storage
  localStorage.removeItem('access_token');
  localStorage.removeItem('refresh_token');
  localStorage.removeItem('id_token');

  // Redirect to login page
  window.location.href = '/login';
}

// Revoke specific access token (e.g., when disconnecting a device)
async function revokeAccessToken(accessToken) {
  await fetch('https://auth.example.com/connect/revoke', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/x-www-form-urlencoded'
    },
    body: new URLSearchParams({
      token: accessToken,
      token_type_hint: 'access_token',
      client_id: 'my-client'
    })
  });
}

Python

python 
import requests

def revoke_token(token: str, token_type: str = 'refresh_token') -> bool:
    """Revoke an OAuth token.

    Args:
        token: The token to revoke
        token_type: Either 'access_token' or 'refresh_token'

    Returns:
        True if the request succeeded (token may or may not have existed)
    """
    response = requests.post(
        'https://auth.example.com/connect/revoke',
        data={
            'token': token,
            'token_type_hint': token_type,
            'client_id': 'my-client',
            'client_secret': 'my-secret'  # For confidential clients
        }
    )
    return response.status_code == 200

def logout(refresh_token: str):
    """Perform a complete logout by revoking the refresh token."""
    revoke_token(refresh_token, 'refresh_token')
    # Clear session data
    session.clear()

# Usage
logout(user_refresh_token)

cURL

bash 
# Revoke a refresh token (public client)
curl -X POST https://auth.example.com/connect/revoke \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "token=8xLOxBtZp8" \
  -d "token_type_hint=refresh_token" \
  -d "client_id=my-client"

# Revoke an access token (confidential client)
curl -X POST https://auth.example.com/connect/revoke \
  -u "my-client:my-secret" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "token=eyJhbGciOiJSUzI1NiIsInR5cCI6ImF0K2p3dCJ9..." \
  -d "token_type_hint=access_token"

Error Responses

Errors are only returned for client authentication failures or malformed requests:

401 Unauthorized

json 
{
  "error": "invalid_client",
  "error_description": "Client authentication failed"
}

400 Bad Request

json 
{
  "error": "invalid_request",
  "error_description": "The 'token' parameter is required"
}

Common Use Cases

1. User Logout

Revoke the refresh token when the user clicks "Sign Out" to invalidate the entire session.

2. Device Disconnection

When a user wants to disconnect a specific device or MCP client, revoke that device's tokens.

3. Security Response

If suspicious activity is detected, revoke all tokens for the affected user or client.

4. MCP Integration Removal

When a user disconnects an AI assistant (like Claude Desktop), revoke the tokens issued to that client.
