Token Endpoint

The token endpoint exchanges authorization codes, refresh tokens, or client credentials for access tokens and optionally ID tokens and refresh tokens.

POST /connect/token

Supported Grant Types

Grant Type Description Use Case
authorization_code Exchange auth code for tokens Web apps, mobile apps, MCP clients
refresh_token Exchange refresh token for new tokens Token renewal without user interaction
client_credentials Authenticate as the client itself Server-to-server communication

Authorization Code Grant

Exchange an authorization code for tokens. This is the most common flow for user authentication.

Request Parameters

Parameter Required Description
grant_type Required Must be authorization_code
code Required The authorization code received from the authorize endpoint
redirect_uri Required Must match the redirect_uri used in the authorization request
client_id Required The client identifier
code_verifier Required* PKCE code verifier. Required if code_challenge was used
client_secret Conditional Required for confidential clients

Request Example

http 
POST /connect/token HTTP/1.1
Host: auth.example.com
Content-Type: application/x-www-form-urlencoded

grant_type=authorization_code&
code=SplxlOBeZQQYbYS6WxSbIA&
redirect_uri=https://app.example.com/callback&
client_id=my-client&
code_verifier=dBjftJeZ4CVP-mB92K27uhbUJU1p1r_wW1gFWFOEjXk
javascript 
const response = await fetch('https://auth.example.com/connect/token', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/x-www-form-urlencoded'
  },
  body: new URLSearchParams({
    grant_type: 'authorization_code',
    code: authorizationCode,
    redirect_uri: 'https://app.example.com/callback',

Refresh Token Grant

Exchange a refresh token for new access and refresh tokens without requiring user interaction.

Request Parameters

Parameter Required Description
grant_type Required Must be refresh_token
refresh_token Required The refresh token
client_id Required The client identifier
scope Optional Request a subset of the original scopes
client_secret Conditional Required for confidential clients

Request Example

http 
POST /connect/token HTTP/1.1
Host: auth.example.com
Content-Type: application/x-www-form-urlencoded

grant_type=refresh_token&
refresh_token=8xLOxBtZp8&
client_id=my-client
javascript 
const response = await fetch('https://auth.example.com/connect/token', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/x-www-form-urlencoded'
  },
  body: new URLSearchParams({
    grant_type: 'refresh_token',
    refresh_token: storedRefreshToken,
    client_id: 'my-client'
  })
});

const newTokens = await response.json();
// Update stored tokens with new values

Success Response

json 
{
  "access_token": "eyJhbGciOiJSUzI1NiIsInR5cCI6ImF0K2p3dCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "refresh_token": "tGzv3JOkF0XG5Qx2TlKWIA",
  "scope": "openid profile email"
}
Refresh Token Rotation

Andy Auth uses refresh token rotation for security. Each time you use a refresh token, a new refresh token is issued and the old one is invalidated. Always store and use the latest refresh token.

Client Credentials Grant

Authenticate as the client itself without a user context. Used for server-to-server communication.

Request Parameters

Parameter Required Description
grant_type Required Must be client_credentials
client_id Required The client identifier
client_secret Required The client secret
scope Optional Space-separated list of requested scopes

Request Example

http 
POST /connect/token HTTP/1.1
Host: auth.example.com
Content-Type: application/x-www-form-urlencoded
Authorization: Basic bXktY2xpZW50Om15LXNlY3JldA==

grant_type=client_credentials&
scope=api:read api:write
javascript 
const credentials = btoa('my-client:my-secret');

const response = await fetch('https://auth.example.com/connect/token', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/x-www-form-urlencoded',
    'Authorization': `Basic ${credentials}`
  },
  body: new URLSearchParams({
    grant_type: 'client_credentials',
    scope: 'api:read api:write'
  })
});

const tokens = await response.json();

Success Response

json 
{
  "access_token": "eyJhbGciOiJSUzI1NiIsInR5cCI6ImF0K2p3dCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "scope": "api:read api:write"
}

Response Fields

Field Type Description
access_token string The access token (JWT)
token_type string Always Bearer
expires_in number Token lifetime in seconds
refresh_token string Refresh token (if offline_access scope was granted)
id_token string ID token (if openid scope was requested)
scope string Space-separated list of granted scopes

Error Response

json 
{
  "error": "invalid_grant",
  "error_description": "The authorization code has expired."
}

Error Codes

Error Description
invalid_request Missing or invalid parameter
invalid_client Client authentication failed
invalid_grant Authorization code or refresh token is invalid, expired, or revoked
unauthorized_client Client not authorized for this grant type
unsupported_grant_type Grant type not supported
invalid_scope Requested scope is invalid or exceeds originally granted scope
