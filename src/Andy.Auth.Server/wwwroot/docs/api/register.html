Dynamic Client Registration

The registration endpoint allows OAuth clients to register dynamically without manual configuration. This implements RFC 7591 (OAuth 2.0 Dynamic Client Registration Protocol).

POST /connect/register
MCP Requirement

Dynamic Client Registration is a key requirement for MCP OAuth 2.1 authentication. It allows AI clients like Claude Desktop, ChatGPT, and VS Code extensions to register automatically when connecting to a new authorization server.

Register a New Client

Request Body

Send a JSON object with the client metadata:

Field Required Description
client_name Required Human-readable name for the client
redirect_uris Required Array of allowed redirect URIs
grant_types Optional Array of grant types. Default: ["authorization_code"]
response_types Optional Array of response types. Default: ["code"]
token_endpoint_auth_method Optional Authentication method. Default: "none" for public clients
scope Optional Space-separated list of allowed scopes
client_uri Optional URL of the client's home page
logo_uri Optional URL of the client's logo
contacts Optional Array of email addresses for responsible parties
tos_uri Optional URL of the client's terms of service
policy_uri Optional URL of the client's privacy policy

Request Example

http 
POST /connect/register HTTP/1.1
Host: auth.example.com
Content-Type: application/json

{
  "client_name": "My MCP Client",
  "redirect_uris": [
    "http://localhost:3000/callback",
    "http://127.0.0.1:3000/callback"
  ],
  "grant_types": ["authorization_code", "refresh_token"],
  "response_types": ["code"],
  "token_endpoint_auth_method": "none",
  "scope": "openid profile email offline_access"
}

Success Response

Returns the registered client information including the generated client_id:

json 
{
  "client_id": "dcr_abc123xyz789",
  "client_name": "My MCP Client",
  "redirect_uris": [
    "http://localhost:3000/callback",
    "http://127.0.0.1:3000/callback"
  ],
  "grant_types": ["authorization_code", "refresh_token"],
  "response_types": ["code"],
  "token_endpoint_auth_method": "none",
  "scope": "openid profile email offline_access",
  "client_id_issued_at": 1704150000,
  "registration_access_token": "rat_xyz789abc123...",
  "registration_client_uri": "https://auth.example.com/connect/register/dcr_abc123xyz789"
}

Response Fields

Field Description
client_id Unique identifier for the registered client
client_secret Client secret (only for confidential clients)
client_id_issued_at Unix timestamp when the client was registered
client_secret_expires_at Unix timestamp when the secret expires (0 = never)
registration_access_token Token for accessing the registration management endpoint
registration_client_uri URI for managing this client registration

Authentication Methods

Method Description Use Case
none Public client (no secret) MCP clients, mobile apps, SPAs
client_secret_basic Secret via HTTP Basic Auth Server-side applications
client_secret_post Secret in request body Server-side applications

Code Examples

JavaScript - MCP Client Registration

javascript 
async function registerClient(authServerUrl, clientName, redirectUri) {
  // First, discover the registration endpoint
  const discoveryResponse = await fetch(
    `${authServerUrl}/.well-known/openid-configuration`
  );
  const discovery = await discoveryResponse.json();

  if (!discovery.registration_endpoint) {
    throw new Error('Server does not support dynamic client registration');
  }

  // Register the client
  const response = await fetch(discovery.registration_endpoint, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      client_name: clientName,
      redirect_uris: [redirectUri],
      grant_types: ['authorization_code', 'refresh_token'],
      response_types: ['code'],
      token_endpoint_auth_method: 'none',
      scope: 'openid profile email offline_access'
    })
  });

  if (!response.ok) {
    const error = await response.json();
    throw new Error(`Registration failed: ${error.error_description || error.error}`);
  }

  const clientInfo = await response.json();

  // Store the client credentials securely
  return {
    clientId: clientInfo.client_id,
    registrationAccessToken: clientInfo.registration_access_token,
    registrationUri: clientInfo.registration_client_uri
  };
}

// Usage
const client = await registerClient(
  'https://auth.example.com',
  'My MCP Tool',
  'http://localhost:3000/callback'
);
console.log('Registered client ID:', client.clientId);

Python

python 
import requests

def register_mcp_client(auth_server_url: str, client_name: str, redirect_uri: str) -> dict:
    """Register a new MCP client dynamically."""

    # Discover endpoints
    discovery = requests.get(
        f'{auth_server_url}/.well-known/openid-configuration'
    ).json()

    registration_endpoint = discovery.get('registration_endpoint')
    if not registration_endpoint:
        raise ValueError('Server does not support dynamic client registration')

    # Register the client
    response = requests.post(
        registration_endpoint,
        json={
            'client_name': client_name,
            'redirect_uris': [redirect_uri],
            'grant_types': ['authorization_code', 'refresh_token'],
            'response_types': ['code'],
            'token_endpoint_auth_method': 'none',
            'scope': 'openid profile email offline_access'
        }
    )

    if not response.ok:
        error = response.json()
        raise ValueError(f"Registration failed: {error.get('error_description', error.get('error'))}")

    return response.json()

# Usage
client_info = register_mcp_client(
    'https://auth.example.com',
    'My Python MCP Client',
    'http://localhost:8000/callback'
)
print(f"Client ID: {client_info['client_id']}")

Managing Client Registration

After registration, you can use the registration_access_token and registration_client_uri to read, update, or delete the client registration.

Read Client Configuration

GET /connect/register/{client_id}
http 
GET /connect/register/dcr_abc123xyz789 HTTP/1.1
Host: auth.example.com
Authorization: Bearer rat_xyz789abc123...

Update Client Configuration

PUT /connect/register/{client_id}
http 
PUT /connect/register/dcr_abc123xyz789 HTTP/1.1
Host: auth.example.com
Authorization: Bearer rat_xyz789abc123...
Content-Type: application/json

{
  "client_id": "dcr_abc123xyz789",
  "client_name": "Updated Client Name",
  "redirect_uris": [
    "http://localhost:3000/callback",
    "http://localhost:4000/callback"
  ],
  "grant_types": ["authorization_code", "refresh_token"],
  "response_types": ["code"],
  "token_endpoint_auth_method": "none"
}

Delete Client Registration

DELETE /connect/register/{client_id}
http 
DELETE /connect/register/dcr_abc123xyz789 HTTP/1.1
Host: auth.example.com
Authorization: Bearer rat_xyz789abc123...

400 Bad Request

json 
{
  "error": "invalid_client_metadata",

Error Description

invalid_redirect_uri One or more redirect URIs are invalid
invalid_software_statement Software statement JWT is invalid (if provided)
unapproved_software_statement Software statement is not approved
