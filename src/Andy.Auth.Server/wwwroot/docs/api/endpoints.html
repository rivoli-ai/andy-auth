API Endpoints Overview

Andy Auth implements a complete OAuth 2.0 and OpenID Connect authorization server using OpenIddict. This reference documents all available API endpoints.

Base URL

All endpoints are relative to your Andy Auth server URL. For example, if your server is at https://auth.example.com, the authorization endpoint would be https://auth.example.com/connect/authorize.

OAuth 2.0 / OIDC Endpoints

These are the core endpoints for authentication and authorization:

Endpoint Method Description
/connect/authorize GET POST Initiates the OAuth authorization flow
/connect/token POST Exchanges authorization codes or credentials for tokens
/connect/userinfo GET Returns claims about the authenticated user
/connect/introspect POST Validates and returns information about a token
/connect/revoke POST Revokes an access or refresh token
/connect/register POST Dynamic Client Registration (RFC 7591)

Discovery Endpoints

Standard endpoints for discovering server capabilities:

Endpoint Method Description
/.well-known/openid-configuration GET OpenID Connect Discovery document
/.well-known/jwks GET JSON Web Key Set for token verification

Authentication Methods

Andy Auth supports the following client authentication methods:

Method Description Use Case
none No client authentication (public clients) Mobile apps, SPAs, MCP clients
client_secret_basic HTTP Basic authentication Server-side applications
client_secret_post Client secret in request body Server-side applications

Error Responses

All endpoints return standard OAuth 2.0 error responses:

json 
{
  "error": "invalid_request",
  "error_description": "The 'client_id' parameter is missing."
}

Common error codes:

Error Code Description
invalid_request The request is missing a required parameter or is malformed
invalid_client Client authentication failed
invalid_grant The authorization code or refresh token is invalid or expired
unauthorized_client The client is not authorized to use this grant type
unsupported_grant_type The grant type is not supported
invalid_scope The requested scope is invalid or unknown
access_denied The resource owner denied the request
