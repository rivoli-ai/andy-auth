Introspection Endpoint

The introspection endpoint allows resource servers to validate access tokens and retrieve their metadata. This implements RFC 7662 (OAuth 2.0 Token Introspection).

POST /connect/introspect
MCP Token Validation

The introspection endpoint is essential for MCP servers to validate tokens from AI clients. When an MCP server receives a request with an access token, it should call this endpoint to verify the token is valid and get user information.

Request

Request Parameters

Parameter Required Description
token Required The token to introspect (access token or refresh token)
token_type_hint Optional Hint about the token type: access_token or refresh_token

Client Authentication

Introspection requests must be authenticated. Andy Auth supports:

  • HTTP Basic Authentication - Recommended for confidential clients
  • POST body credentials - client_id and client_secret in the request body

Request Example

http 
POST /connect/introspect HTTP/1.1
Host: auth.example.com
Content-Type: application/x-www-form-urlencoded
Authorization: Basic bXktY2xpZW50Om15LXNlY3JldA==

token=eyJhbGciOiJSUzI1NiIsInR5cCI6ImF0K2p3dCJ9...&
token_type_hint=access_token

Active Token Response

When the token is valid and active:

json 
{
  "active": true,
  "sub": "user_abc123",
  "client_id": "my-client",
  "username": "johndoe",
  "token_type": "Bearer",
  "scope": "openid profile email",
  "aud": "my-api",
  "iss": "https://auth.example.com/",
  "exp": 1704153600,
  "iat": 1704150000,
  "nbf": 1704150000,
  "jti": "token_xyz789"
}

Response Fields

Field Type Description
active boolean Always present. Indicates if the token is currently active
sub string Subject identifier (user ID)
client_id string Client that requested the token
username string Human-readable username
token_type string Type of token (usually "Bearer")
scope string Space-separated list of scopes
aud string Intended audience(s)
iss string Issuer URL
exp number Expiration timestamp (Unix epoch)
iat number Issued at timestamp (Unix epoch)
nbf number Not before timestamp (Unix epoch)
jti string Unique token identifier

Inactive Token Response

When the token is invalid, expired, revoked, or unknown:

json 
{
  "active": false
}
Security Note

For security reasons, the introspection endpoint returns only {"active": false} for invalid tokens, without revealing why the token is invalid. This prevents attackers from learning about token validity or structure.

Code Examples

Python (MCP Server)

python 
import requests
from functools import wraps
from flask import request, jsonify

INTROSPECTION_URL = 'https://auth.example.com/connect/introspect'
CLIENT_ID = 'my-mcp-server'
CLIENT_SECRET = 'my-secret'

def validate_token(access_token: str) -> dict | None:
    """Validate an access token using introspection."""
    response = requests.post(
        INTROSPECTION_URL,
        data={
            'token': access_token,
            'token_type_hint': 'access_token'
        },
        auth=(CLIENT_ID, CLIENT_SECRET)
    )

    if response.status_code != 200:
        return None

    result = response.json()
    if not result.get('active'):
        return None

    return result

def require_auth(f):
    """Decorator to require valid OAuth token."""
    @wraps(f)
    def decorated(*args, **kwargs):
        auth_header = request.headers.get('Authorization', '')
        if not auth_header.startswith('Bearer '):
            return jsonify({'error': 'Missing bearer token'}), 401

        token = auth_header[7:]  # Remove 'Bearer ' prefix
        token_info = validate_token(token)

        if not token_info:
            return jsonify({'error': 'Invalid or expired token'}), 401

        # Make token info available to the route
        request.token_info = token_info
        return f(*args, **kwargs)

    return decorated

# Usage in an MCP server endpoint
@app.route('/mcp/tools', methods=['POST'])
@require_auth
def handle_mcp_request():
    user_id = request.token_info['sub']
    scopes = request.token_info.get('scope', '').split()

    # Check for required scope
    if 'mcp:tools' not in scopes:
        return jsonify({'error': 'Insufficient scope'}), 403

    # Process MCP request...
    return jsonify({'result': 'success'})

JavaScript / Node.js

javascript 
const INTROSPECTION_URL = 'https://auth.example.com/connect/introspect';
const CLIENT_ID = 'my-mcp-server';
const CLIENT_SECRET = 'my-secret';

async function validateToken(accessToken) {
  const credentials = Buffer.from(`${CLIENT_ID}:${CLIENT_SECRET}`).toString('base64');

  const response = await fetch(INTROSPECTION_URL, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/x-www-form-urlencoded',
      'Authorization': `Basic ${credentials}`
    },
    body: new URLSearchParams({
      token: accessToken,
      token_type_hint: 'access_token'
    })
  });

  if (!response.ok) {
    return null;
  }

  const result = await response.json();
  return result.active ? result : null;
}

// Express middleware
function requireAuth(req, res, next) {
  const authHeader = req.headers.authorization || '';
  if (!authHeader.startsWith('Bearer ')) {
    return res.status(401).json({ error: 'Missing bearer token' });
  }

  const token = authHeader.slice(7);
  validateToken(token)
    .then(tokenInfo => {
      if (!tokenInfo) {
        return res.status(401).json({ error: 'Invalid or expired token' });
      }
      req.tokenInfo = tokenInfo;
      next();
    })
    .catch(err => {
      console.error('Token validation error:', err);
      res.status(500).json({ error: 'Token validation failed' });
    });
}

// Usage
app.post('/mcp/tools', requireAuth, (req, res) => {
  const userId = req.tokenInfo.sub;
  const scopes = (req.tokenInfo.scope || '').split(' ');

  // Check scope and process request...
  res.json({ result: 'success' });
});

Error Responses

401 Unauthorized

Returned when client authentication fails:

json 
{
  "error": "invalid_client",
  "error_description": "Client authentication failed"
}

400 Bad Request

Returned when the request is malformed:

json 
{
  "error": "invalid_request",
  "error_description": "The 'token' parameter is required"
}

Caching Considerations

Introspection requests can add latency to API calls. Consider these strategies to reduce overhead:

  • Short-term caching - Cache introspection results for a few seconds to handle bursts of requests
  • JWT validation - For JWTs, validate the signature locally first, then introspect periodically
  • Token binding - Use the jti claim as a cache key
Cache Carefully

If you cache introspection results, keep the cache duration short (seconds, not minutes). Longer cache times may allow revoked tokens to continue working.
