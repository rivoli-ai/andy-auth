Security Overview

Andy Auth is built with security as a top priority, implementing industry-standard protocols and best practices to protect your authentication infrastructure.

Security by Design

Andy Auth follows the principle of security by design, implementing multiple layers of protection that work together to safeguard your users and applications.

OAuth 2.1 Compliance

Andy Auth implements the OAuth 2.1 specification, which consolidates best practices and security improvements over OAuth 2.0:

  • PKCE is mandatory for all authorization code flows
  • Implicit grant removed - no longer supported due to security concerns
  • Resource Owner Password grant removed - deprecated for security reasons
  • Bearer tokens only over HTTPS - enforced in production
  • Refresh token rotation - automatic rotation on use
text 
OAuth 2.1 Security Improvements:
+----------------------------------+-------------------+
| Feature                          | OAuth 2.1         |
+----------------------------------+-------------------+
| PKCE                             | Required          |
| Implicit Grant                   | Removed           |
| Password Grant                   | Removed           |
| Refresh Token Rotation           | Recommended       |
| Exact Redirect URI Matching      | Required          |
+----------------------------------+-------------------+

PKCE Requirement

Proof Key for Code Exchange (PKCE) is required for all public clients. This prevents authorization code interception attacks:

javascript 
// PKCE flow - required for public clients
const codeVerifier = generateCryptographicallySecureString(64);
const codeChallenge = base64UrlEncode(sha256(codeVerifier));

// Include in authorization request
const authUrl = new URL('/connect/authorize', authServer);
authUrl.searchParams.set('code_challenge', codeChallenge);
authUrl.searchParams.set('code_challenge_method', 'S256');

// Include verifier in token exchange
const tokenRequest = {
    grant_type: 'authorization_code',
    code: authorizationCode,
    code_verifier: codeVerifier  // Proves possession of original challenge
};
S256 Only

Andy Auth only supports the S256 code challenge method. The plain method is rejected for security reasons.

Token Security

Andy Auth implements comprehensive token security measures:

Token Rotation

Refresh tokens are automatically rotated on each use. When a refresh token is used to obtain a new access token, the old refresh token is invalidated and a new one is issued:

json 
// Token refresh response with rotation
{
    "access_token": "new_access_token_xyz",
    "token_type": "Bearer",
    "expires_in": 3600,
    "refresh_token": "new_refresh_token_abc",  // New refresh token
    "scope": "openid profile email"
}

Token Expiration

Default token lifetimes are configured for security:

Token Type Default Lifetime Configurable
Access Token 1 hour Yes
Refresh Token 14 days Yes
Authorization Code 5 minutes No
ID Token 1 hour Yes

Token Revocation

Tokens can be revoked immediately using the revocation endpoint:

POST /connect/revoke

Rate Limiting

Andy Auth includes built-in rate limiting to protect against brute force attacks and API abuse:

json 
// Rate limiting configuration in appsettings.json
{
    "RateLimiting": {
        "EnableRateLimiting": true,
        "PermitLimit": 100,
        "WindowInSeconds": 60,
        "QueueLimit": 10,
        "TokenEndpointPermitLimit": 20,
        "TokenEndpointWindowInSeconds": 60,
        "LoginPermitLimit": 5,
        "LoginWindowInSeconds": 300
    }
}
Endpoint Default Limit Window
General API 100 requests 1 minute
Token Endpoint 20 requests 1 minute
Login Endpoint 5 attempts 5 minutes
Registration 3 requests 1 hour

Two-Factor Authentication

Andy Auth supports two-factor authentication (2FA) using TOTP (Time-based One-Time Passwords):

  • TOTP Support - Compatible with Google Authenticator, Authy, and other TOTP apps
  • Recovery Codes - One-time use backup codes for account recovery
  • Per-User Enforcement - Administrators can require 2FA for specific users or roles
  • Remember Device - Optional trusted device support

Audit Logging

All security-relevant events are logged for compliance and forensic analysis:

Event Category Events Logged
Authentication Login success/failure, logout, password changes
Authorization Token issuance, consent grants, scope changes
Client Management Client registration, updates, deletions
User Management Account creation, role changes, 2FA enrollment
Security Events Rate limit violations, suspicious activity

Audit logs include:

  • Timestamp (UTC)
  • User ID and IP address
  • Action performed
  • Success/failure status
  • Additional context data

CORS Configuration

Cross-Origin Resource Sharing (CORS) is configured to control which origins can access the API:

json 
// CORS configuration in appsettings.json
{
    "Cors": {
        "AllowedOrigins": [
            "https://app.example.com",
            "https://admin.example.com"
        ],
        "AllowedMethods": ["GET", "POST"],
        "AllowedHeaders": ["Authorization", "Content-Type"],
        "AllowCredentials": true,
        "MaxAge": 86400
    }
}
Production Warning

Never use wildcard (*) origins in production. Always explicitly list allowed origins to prevent unauthorized cross-origin requests.

Security Headers

Andy Auth automatically sets security headers on all responses:

Header Value Purpose
Strict-Transport-Security max-age=31536000; includeSubDomains Force HTTPS connections
X-Content-Type-Options nosniff Prevent MIME type sniffing
X-Frame-Options DENY Prevent clickjacking
X-XSS-Protection 1; mode=block Enable browser XSS filter
Content-Security-Policy Restrictive policy Prevent XSS and data injection
Referrer-Policy strict-origin-when-cross-origin Control referrer information

Security Best Practices

Follow these recommendations for maximum security:

  • Always use HTTPS in production environments
  • Rotate secrets regularly - encryption keys, signing keys, client secrets
  • Enable 2FA for all administrator accounts
  • Review audit logs regularly for suspicious activity
  • Keep Andy Auth updated to receive security patches
  • Use strong passwords and enforce password policies
  • Limit token lifetimes based on your security requirements
  • Restrict CORS origins to only necessary domains
