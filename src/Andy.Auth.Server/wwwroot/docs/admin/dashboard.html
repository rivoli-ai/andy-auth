Admin Dashboard Overview

The Andy Auth Admin Dashboard provides a centralized interface for managing users, OAuth clients, tokens, and monitoring system activity. Access it at /admin.

Admin Access Required

The admin dashboard is only accessible to users with administrator privileges. Contact your system administrator if you need access.

Accessing the Dashboard

To access the admin dashboard:

  1. Navigate to https://your-domain.com/admin
  2. Log in with an administrator account
  3. You will be redirected to the dashboard overview
GET /admin

Dashboard Statistics

The dashboard home page displays key system metrics at a glance:

Dashboard Statistics Screenshot
Metric Description
Total Users The total number of registered users in the system
Active Users Users who have logged in within the configured activity period
OAuth Clients The number of registered OAuth 2.0/OIDC client applications

Recent Login Activity

The dashboard displays a list of recent user logins, showing:

  • Email - The user's email address
  • Last Login - The date and time of their most recent login

This helps administrators quickly identify active users and detect any unusual login patterns.

Quick Actions

The dashboard provides convenient shortcuts to common administrative tasks:

Manage OAuth Clients
Create, edit, and delete OAuth client applications.
Manage Users
View, suspend, and manage user accounts.

Dashboard Navigation

The admin interface is organized into the following sections:

Section Path Description
Dashboard /admin Overview with statistics and recent activity
Users /admin/users User account management
Clients /admin/clients OAuth client application management
Tokens /admin/tokens Active token viewing and revocation
Audit Logs /admin/auditlogs Security and administrative action logs

Security Considerations

Security Best Practices

  • Limit admin access to only those who require it
  • Use strong, unique passwords for admin accounts
  • Enable two-factor authentication (2FA) for all admin users
  • Regularly review audit logs for suspicious activity
  • Log out of admin sessions when not in use
