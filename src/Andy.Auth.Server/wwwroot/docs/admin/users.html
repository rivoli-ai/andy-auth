User Management

The User Management interface allows administrators to view, search, and manage all user accounts in the system. Access it at /admin/users.

GET /admin/users

User List Overview

The user list displays all registered users with the following information:

User List Screenshot
Column Description
Email User's email address with verification status badge
Full Name User's display name (if set)
Role User role - either "Admin" or "User"
Status Account status: Active, Inactive, Suspended, or Deleted
Expiration Account expiration date (if set)
Created Account creation date
Last Login Most recent login timestamp

Searching and Filtering

Use the search bar to find users by email or name:

  1. Enter a search term in the search input field
  2. Click Search or press Enter
  3. Click Clear to reset the search

The user list also supports sorting by clicking on column headers:

  • Email - Sort alphabetically by email
  • Full Name - Sort alphabetically by name
  • Created - Sort by account creation date
  • Last Login - Sort by most recent login

Understanding User Status

User Status Types 
Active      - User can log in and access the system
Inactive    - User has not confirmed their email
Suspended   - User has been temporarily disabled by an admin
Deleted     - User has been soft-deleted (cannot be restored)

User Actions

Each user row provides action buttons for managing that account:

Suspending a User

To temporarily disable a user account:

  1. Click the Suspend button (warning icon) next to the user
  2. Enter a reason for the suspension in the modal dialog
  3. Click Suspend User to confirm
Suspension Notes

Suspended users cannot log in until an administrator unsuspends their account. The suspension reason is recorded and visible in the audit logs.

Unsuspending a User

To restore access for a suspended user:

  1. Click the Unsuspend button (checkmark icon) next to the suspended user
  2. Confirm the action in the prompt

Setting Account Expiration

To set a date when a user account will automatically expire:

  1. Click the Set Expiration button (calendar icon)
  2. Select an expiration date from the date picker
  3. Click Set Expiration to save

Leave the date field empty to remove an existing expiration.

Editing User Name

To update a user's display name:

  1. Click the Edit Name button (pencil icon)
  2. Enter the new name in the modal dialog
  3. Click Update Name to save

Resetting User Password

To reset a user's password:

  1. Click the Reset Password button (lock icon)
  2. Enter the new password in the modal dialog
  3. Click Reset Password to apply
Password Requirements

Passwords must be at least 8 characters and include uppercase letters, lowercase letters, and numbers. Consider informing the user of their new password through a secure channel.

Deleting a User

To permanently remove a user account:

  1. Click the Delete button (trash icon)
  2. Review the warning message
  3. Click Delete User to confirm
Deletion Warning

User deletion is a soft-delete operation. The user record is marked as deleted but retained in the database. System users cannot be deleted.

Email Verification Status

Users with verified email addresses display a "Verified" badge next to their email. Unverified users may have limited access depending on your system configuration.

Admin Role Indicator

Users with administrator privileges display an "Admin" badge in the Role column. Admin users have access to the admin dashboard and can manage other users.

Pagination

The user list is paginated to improve performance with large numbers of users. Use the pagination controls at the bottom of the list to navigate between pages.

Best Practices

  • Regular Reviews - Periodically review user accounts to identify inactive or suspicious accounts
  • Document Suspensions - Always provide clear reasons when suspending accounts
  • Expiration Dates - Use account expiration for temporary access or contractors
  • Secure Password Distribution - When resetting passwords, use secure methods to communicate new credentials
  • Audit Trail - All user management actions are logged in the audit logs
