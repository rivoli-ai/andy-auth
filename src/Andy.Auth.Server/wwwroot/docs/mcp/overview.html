MCP Integration Overview

Andy Auth provides native support for the Model Context Protocol (MCP) OAuth 2.1 specification, enabling secure authentication for AI assistants and tools.

What is MCP?

The Model Context Protocol (MCP) is an open standard for connecting AI assistants to external data sources and tools. MCP uses OAuth 2.1 for authentication, ensuring secure access to protected resources.

Supported AI Clients

Andy Auth includes pre-configured OAuth clients for popular AI assistants:

Client Client ID Type Use Case
Claude Desktop claude-desktop Public Anthropic's Claude Desktop application
ChatGPT chatgpt Public OpenAI's ChatGPT custom actions
Cline cline Public Cline VS Code extension
Roo roo Public Roo VS Code extension
Continue.dev continue-dev Public Continue.dev VS Code/JetBrains extension

OAuth 2.1 for MCP

MCP authentication uses OAuth 2.1 with the following key requirements:

  • Authorization Code Flow with PKCE - Required for all public clients
  • Dynamic Client Registration (RFC 7591) - Allows clients to register automatically
  • Token Introspection - For validating tokens on MCP servers
  • Token Revocation - For secure logout

GET /.well-known/openid-configuration

Dynamic Client Registration

MCP clients can register automatically using RFC 7591 Dynamic Client Registration:

POST /connect/register
json 
// Request
POST /connect/register HTTP/1.1
Content-Type: application/json

{
  "client_name": "My MCP Client",
  "redirect_uris": ["http://localhost:3000/callback"],
  "grant_types": ["authorization_code", "refresh_token"],
  "response_types": ["code"],
  "token_endpoint_auth_method": "none"
}

// Response
{
  "client_id": "dcr_abc123xyz",
  "client_name": "My MCP Client",
  "redirect_uris": ["http://localhost:3000/callback"],
  "grant_types": ["authorization_code", "refresh_token"],
  "registration_access_token": "rat_...",
  "registration_client_uri": "https://auth.example.com/connect/register/dcr_abc123xyz"
}

PKCE Requirement

All MCP clients must use PKCE (Proof Key for Code Exchange) for security:

javascript 
// Generate PKCE values
const codeVerifier = generateRandomString(64);
const codeChallenge = base64UrlEncode(sha256(codeVerifier));

// Authorization request
const authUrl = new URL('https://auth.example.com/connect/authorize');
authUrl.searchParams.set('client_id', 'my-client');
authUrl.searchParams.set('response_type', 'code');
authUrl.searchParams.set('redirect_uri', 'http://localhost:3000/callback');
authUrl.searchParams.set('scope', 'openid profile email');
authUrl.searchParams.set('code_challenge', codeChallenge);
authUrl.searchParams.set('code_challenge_method', 'S256');
authUrl.searchParams.set('state', generateRandomString(32));

// Token exchange (include code_verifier)
const tokenResponse = await fetch('https://auth.example.com/connect/token', {
  method: 'POST',
  headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
  body: new URLSearchParams({
    grant_type: 'authorization_code',
    client_id: 'my-client',
    code: authorizationCode,
    redirect_uri: 'http://localhost:3000/callback',
    code_verifier: codeVerifier  // Include the original verifier
  })
});

Token Validation on MCP Servers

MCP servers should validate tokens using the introspection endpoint:

python 
import requests

def validate_token(access_token: str) -> dict:
    """Validate an access token using introspection."""
    response = requests.post(
        'https://auth.example.com/connect/introspect',
        data={
            'token': access_token,
            'token_type_hint': 'access_token'
        },
        auth=('my-client-id', 'my-client-secret')  # For confidential clients
    )

    result = response.json()

    if not result.get('active'):
        raise ValueError('Token is not active')

    return {
        'user_id': result.get('sub'),
        'email': result.get('email'),
        'scopes': result.get('scope', '').split()
    }
