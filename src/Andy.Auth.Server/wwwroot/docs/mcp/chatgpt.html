ChatGPT Integration

Configure OpenAI's ChatGPT custom actions to authenticate with Andy Auth, enabling secure access to your MCP-enabled APIs and services.

Pre-configured Client

Andy Auth includes a pre-configured OAuth client for ChatGPT with the client ID chatgpt. This works with ChatGPT Plus, Team, and Enterprise plans that support custom GPTs and actions.

Prerequisites

  • ChatGPT Plus, Team, or Enterprise subscription
  • Access to create custom GPTs
  • Andy Auth server with a publicly accessible URL (HTTPS required)
  • An API or MCP server to integrate

Step 1: Create a Custom GPT

  1. Navigate to chat.openai.com
  2. Click on your profile icon and select "My GPTs"
  3. Click "Create a GPT"
  4. Configure your GPT's name, description, and instructions

Step 2: Configure an Action

In the GPT builder, navigate to the "Configure" tab and scroll to "Actions":

  1. Click "Create new action"
  2. You'll need to provide an OpenAPI schema for your API

OpenAPI Schema Example

yaml 
openapi: 3.1.0
info:
  title: My Company API
  description: API for accessing company data through ChatGPT
  version: 1.0.0

      operationId: getData

      parameters:

          in: query
          required: true
          schema:
            type: string
          description: Search query
      responses:
        '200':
          description: Successful response
          content:
            application/json:
              schema:
                type: object
                properties:
                  results:
                    type: array
                    items:

          description: Unauthorized

    get:
      operationId: getDocuments
      summary: List documents

      responses:

                  properties:
                    id:
                      type: string
                    title:
                      type: string
                    createdAt:
                      type: string
                      format: date-time

Step 3: Configure OAuth Authentication

In the action configuration, select "Authentication" and choose "OAuth":

text 
Authentication Type: OAuth

Client ID: chatgpt

Client Secret: (leave empty for public clients)

Authorization URL: https://auth.example.com/connect/authorize

Token URL: https://auth.example.com/connect/token

Scope: openid profile email offline_access

Token Exchange Method: POST request

OAuth Settings Reference

Setting Value Description
Client ID chatgpt Pre-configured client in Andy Auth
Client Secret (empty) Not required for public clients
Authorization URL https://auth.example.com/connect/authorize Andy Auth authorization endpoint
Token URL https://auth.example.com/connect/token Andy Auth token endpoint
Scope openid profile email Requested permissions

Step 4: Configure Callback URL in Andy Auth

ChatGPT uses a specific callback URL format. Ensure your Andy Auth client configuration includes the ChatGPT callback URL:

text 
https://chat.openai.com/aip/{plugin-id}/oauth/callback
Callback URL Note

The exact callback URL is shown in the ChatGPT action configuration after you save the OAuth settings. Copy this URL and ensure it's added to the allowed redirect URIs in Andy Auth.

The pre-configured chatgpt client in Andy Auth typically allows these redirect patterns:

json 
{
  "client_id": "chatgpt",
  "redirect_uris": [
    "https://chat.openai.com/aip/*/oauth/callback",
    "https://chatgpt.com/aip/*/oauth/callback"
  ]
}

Step 5: User Authentication Flow

When a user interacts with your GPT and triggers an action:

  1. ChatGPT detects the action requires authentication
  2. User clicks "Sign in" button in the chat
  3. Browser popup opens to Andy Auth login page
  4. User authenticates and grants consent
  5. ChatGPT receives the access token
  6. Action executes with the authenticated token

Step 6: Validate Tokens in Your API

Your API should validate the access tokens sent by ChatGPT:

python 
from flask import Flask, request, jsonify
import requests

app = Flask(__name__)

ANDY_AUTH_INTROSPECTION_URL = "https://auth.example.com/connect/introspect"

def validate_token(token):
    """Validate access token via Andy Auth introspection."""
    response = requests.post(
        ANDY_AUTH_INTROSPECTION_URL,
        data={
            "token": token,
            "token_type_hint": "access_token"
        }
    )
    result = response.json()
    return result if result.get("active") else None

@app.route("/api/data")
def get_data():
    # Extract token from Authorization header
    auth_header = request.headers.get("Authorization", "")
    if not auth_header.startswith("Bearer "):
        return jsonify({"error": "Missing bearer token"}), 401

    token = auth_header[7:]  # Remove "Bearer " prefix

    # Validate token
    token_info = validate_token(token)
    if not token_info:
        return jsonify({"error": "Invalid or expired token"}), 401

    # Token is valid - get user info
    user_id = token_info.get("sub")
    email = token_info.get("email")

    # Fetch and return data for this user
    data = fetch_user_data(user_id)
    return jsonify({"results": data})

if __name__ == "__main__":
    app.run(port=5000)

Step 7: Privacy Policy & Terms

ChatGPT requires custom actions to have a privacy policy. Add these URLs to your action configuration:

text 
Privacy Policy URL: https://example.com/privacy
Terms of Service URL: https://example.com/terms

Testing Your Integration

  1. Save your GPT configuration
  2. Open a new chat with your custom GPT
  3. Ask the GPT to perform an action that requires API access
  4. Click "Sign in" when prompted
  5. Complete authentication in the popup
  6. Verify the action executes successfully

Troubleshooting

OAuth Error During Sign-in

Common Causes
  • Invalid redirect URI: Ensure the ChatGPT callback URL is registered in Andy Auth
  • HTTPS required: Andy Auth must be accessible via HTTPS for ChatGPT
  • Client ID mismatch: Verify you're using chatgpt as the client ID

"Action Failed" After Sign-in

Common Causes
  • API not responding: Check your API server is running and accessible
  • Token validation failed: Verify introspection endpoint is working
  • CORS issues: Ensure your API allows requests from ChatGPT's origins
  • OpenAPI schema mismatch: Verify paths match your actual API

Sign-in Popup Blocked

Solution
  • Disable popup blockers for chat.openai.com
  • Allow popups in your browser settings
  • Try a different browser

Security Best Practices

  • Scope limitation: Request only the scopes your GPT needs
  • Token validation: Always validate tokens server-side
  • Rate limiting: Implement rate limits on your API
  • Audit logging: Log all API access for security monitoring
  • Input validation: Sanitize all inputs from ChatGPT
