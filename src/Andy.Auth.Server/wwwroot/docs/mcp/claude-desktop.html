Claude Desktop Integration

Configure Anthropic's Claude Desktop application to authenticate with MCP servers using Andy Auth as your OAuth 2.1 provider.

Pre-configured Client

Andy Auth includes a pre-configured OAuth client for Claude Desktop with the client ID claude-desktop. No additional registration is required.

Prerequisites

  • Claude Desktop installed on your machine (macOS or Windows)
  • Andy Auth server running and accessible
  • An MCP server you want to connect to
  • A user account on Andy Auth

Step 1: Locate Configuration File

Claude Desktop stores its MCP configuration in a JSON file. The location varies by operating system:

Operating System Configuration Path
macOS ~/Library/Application Support/Claude/claude_desktop_config.json
Windows %APPDATA%\Claude\claude_desktop_config.json

Create the file if it doesn't exist:

bash 
# macOS
mkdir -p ~/Library/Application\ Support/Claude
touch ~/Library/Application\ Support/Claude/claude_desktop_config.json

# Windows (PowerShell)
New-Item -ItemType Directory -Force -Path "$env:APPDATA\Claude"
New-Item -ItemType File -Force -Path "$env:APPDATA\Claude\claude_desktop_config.json"

Step 2: Configure MCP Server with OAuth

Add your MCP server configuration with OAuth authentication:

json 
{
  "mcpServers": {
    "my-secure-server": {
      "command": "npx",
      "args": ["-y", "@my-org/mcp-server"],
      "oauth": {
        "clientId": "claude-desktop",
        "authorizationEndpoint": "https://auth.example.com/connect/authorize",
        "tokenEndpoint": "https://auth.example.com/connect/token",
        "scopes": ["openid", "profile", "email", "offline_access"],
        "redirectUri": "http://localhost:8765/callback"
      }
    }
  }
}

Configuration Options

Option Required Description
clientId Yes OAuth client ID (claude-desktop for Andy Auth)
authorizationEndpoint Yes Andy Auth authorization URL
tokenEndpoint Yes Andy Auth token exchange URL
scopes No Array of OAuth scopes to request
redirectUri No Local callback URL for OAuth flow

Alternative: Using OpenID Discovery

Claude Desktop can automatically discover OAuth endpoints:

json 
{
  "mcpServers": {
    "my-secure-server": {
      "command": "npx",
      "args": ["-y", "@my-org/mcp-server"],
      "oauth": {
        "clientId": "claude-desktop",
        "issuer": "https://auth.example.com",
        "scopes": ["openid", "profile", "email", "offline_access"]
      }
    }
  }
}

When using issuer, Claude Desktop fetches /.well-known/openid-configuration to discover all required endpoints automatically.

Step 3: Configure Multiple MCP Servers

You can configure multiple MCP servers, each with their own authentication:

json 
{
  "mcpServers": {
    "company-data": {
      "command": "npx",
      "args": ["-y", "@company/data-mcp-server"],
      "oauth": {
        "clientId": "claude-desktop",
        "issuer": "https://auth.example.com",
        "scopes": ["openid", "profile", "data:read"]
      }
    },
    "code-assistant": {
      "command": "npx",
      "args": ["-y", "@company/code-mcp-server"],
      "oauth": {
        "clientId": "claude-desktop",
        "issuer": "https://auth.example.com",
        "scopes": ["openid", "profile", "code:read", "code:write"]
      }
    },
    "local-files": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/files"]
    }
  }
}

Step 4: Authentication Flow

When you start Claude Desktop and attempt to use an MCP server with OAuth:

  1. Claude Desktop detects the OAuth configuration
  2. A browser window opens to Andy Auth's login page
  3. You authenticate with your username and password
  4. Consent screen shows the requested permissions
  5. After approval, the browser redirects to the local callback
  6. Claude Desktop receives the tokens and connects to the MCP server

Token Management

Claude Desktop manages tokens automatically:

  • Access tokens are cached and reused
  • Refresh tokens automatically obtain new access tokens when expired
  • Token storage is secured by the operating system's keychain
Offline Access

Include offline_access in your scopes to receive refresh tokens. This allows Claude Desktop to maintain authentication across sessions without requiring you to log in again.

Troubleshooting

Browser Doesn't Open

Solution
  • Ensure your default browser is set correctly in system preferences
  • Check if Claude Desktop has permission to open URLs
  • Restart Claude Desktop and try again

Callback Error

Solution
  • Verify the redirectUri is not blocked by a firewall
  • Check that port 8765 (or your configured port) is available
  • Ensure no other application is using the callback port

Token Expired Errors

Solution
  • Add offline_access to your scopes for refresh token support
  • Clear cached tokens: remove OAuth data from your system keychain
  • Re-authenticate by restarting Claude Desktop

Connection Refused to MCP Server

Solution
  • Verify the MCP server is running and accessible
  • Check that the access token has the required scopes
  • Ensure the MCP server is configured to accept tokens from Andy Auth

Viewing Claude Desktop Logs

Access Claude Desktop logs for debugging:

bash 
# macOS
tail -f ~/Library/Logs/Claude/mcp*.log

# Windows (PowerShell)
Get-Content "$env:LOCALAPPDATA\Claude\logs\mcp*.log" -Wait

Security Considerations

  • Localhost callbacks: Claude Desktop uses localhost for OAuth callbacks, which is secure for desktop applications
  • PKCE: Claude Desktop automatically uses PKCE for all OAuth flows
  • Token storage: Tokens are stored securely in the OS keychain
  • Scope limitation: Request only the scopes your MCP servers need
